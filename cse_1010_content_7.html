Objects & O.O.P

In this section, we have learned what Objects are and how to use them to draw. Let's first look at what they are. Objects are very similar to arrays. The differences are how you use them, that objects can store other objects and arrays inside. Let's take a look at how to use them.

  1. var anObject = {
  2. course: "computer science",
  3. level: "2",
  4. completed: {
  5. level1: "CSE1010",
  6. level2: "CSE1110"
  7. }
  8. };

There we go. The code above should now have create an object, called "anObject". Anything inside of the brackets are simply part of the object, much like how an array holds its elements inside square brackets. As you can see though, we have a secodn object inside of the first, the second object is called "completed". It works the exact same way as a normal object, it is simply inside of the first one. Anyways, how would I access those values? Quite simple, simply type the name of the object, a dot and add the element(or the second object if it is nested). A bit like this!

  1. println(anObject.level);
  2. println(anObject.completed.level1);

These should now display the correct values! The first displaying "2" and the second displaying "CSE1010". As you can see, they are quite simple to use! Although, lets now move on to using objects to draw! Let's quickly make two objects, 1 for an adult and 1 for a child...

  1. var childObject = {
  2. name: "Ken",
  3. age: 5,
  4. xPos: 100,
  5. yPos: 100
  6. }
  7. var adultObject = {
  8. name: "Jhon",
  9. age: 23,
  10. xPos: 200,
  11. yPos: 100
  12. }

There we go, we have our two objects. Now, what are those for? Well, as you can see, we mentioned a name, age, and x/y position. This is because we are going to be using those two objects to draw. Let's go ahead and draw these two on the screen...

  1. var drawPerson = function(person) {
  2. fill(255, 0, 0);
  3. var img = getImage("creatures/Winston");
  4. //This image is only avaiable on khan academy, you might have to specify the path of your own when not using it.
  5. image(img, person.xPos, person.yPos);
  6. var txt = person.name + "," + person.age;
  7. text(txt, person.xPos, person.yPos + 20);
  8. };
  10. drawPerson(childObject);
  11. drawPerson(adultObject);

There we go, we should see two different people on the screen now! This is very cool, although, it does seem like we just wasted a lot of our time. What i mean by that is... look at the objects. They are the same thing, the only difference is the values(actual values, not value names) and the names of the object. We just repeated code. That is a horible thing to do when programming. In computer science, there is this acronym. DRY. It stands for, Don't repeat yourself. What did we do? We broke that rule! Although, there is a way to fix this. We can do so by using methods. Methods are essentially like presets. You make a preset and you can create new objects from that. Let's try using those.

  1. var person = function(name, age, xPos, yPos) {
  2. this.name = name;
  3. this.age = age;
  4. this.xPos = xPos;
  5. this.yPos = yPos;
  6. };
  7. // This above is our preset
  8. person.prototype.draw = function() {
  9. fill(255, 0, 0);
  10. var img = getImage("creatures/Winston");
  11. image(img, person.xPos, person.yPos);
  12. var txt = person.name + "," + person.age;
  13. text(txt, person.xPos, person.yPos + 20);
  14. };
  16. var childObject("Ken", 5, 100, 100);
  17. var adultObject("Jhon", 23, 100, 200);
  19. childObject.draw();
  20. adultObject.draw();

There we go. Now, why does this work? Simple, for objects, there is this thing called prototype. This essentially is there to say, use this as a preset. It than creates a function called draw for this preset. What it then does is simply set the values given to the preset and makes a new object out of it. Then the function simply gets the values and uses them. Just that simple. Now, lets say you want them to say something. We can use object inheritance for that. Object inheritance is simply a way to give those methods(or presets) functions they can use. What you do than is you can call it from any object that uses this specific method. Let's see how to use that using a function called talk that we will make.

  1. person.prototype.talk = function() {
  2. text(txt, this.xPos, this.yPos + 10);
  3. };

Now, instead of having the text be in the function and always the same, we can use those methods to change it! To call it, you simple type the objects name followed by a dot and the functions name, in this case it would be like this.

  1. person.talk();

And there we go! This marks off the end of the unit!
